=== HG Financial Consulting — تحديثات قاعدة البيانات (SQL Migrations) ===

الملفات دي بتضيف الجداول/الأعمدة الجديدة على قاعدة البيانات من غير ما تأثّر
على أي بيانات موجودة. كلها آمنة لإعادة التشغيل (بتستخدم IF NOT EXISTS) —
يعني لو اتشغلت قبل كده مش هتعمل أي مشكلة ومش هتمسح حاجة.

────────────────────────────────────────────────────
قبل ما تبدأ (مهم):
- اعمل نسخة احتياطية (Backup / Branch / Snapshot) لقاعدة البيانات من Neon.
- البيانات بتاعتك كلها في قاعدة البيانات، مش في ملفات الموقع.
────────────────────────────────────────────────────

طريقة التشغيل:
1. ادخل على لوحة Neon → SQL Editor (أو أي أداة PostgreSQL بتستخدمها).
2. افتح كل ملف وانسخ محتواه وشغّله — أو شغّلهم بالترتيب ده:

   1) migrate-ai-content-studio.sql
      → أعمدة الجدولة (status / scheduled_at) في المقالات + جدول social_posts

   2) migrate-social-autoposting.sql
      → أعمدة نتيجة النشر + جدول social_credentials (مفاتيح السوشيال المشفّرة)

   3) migrate-case-studies.sql
      → جدول دراسات الحالة (case_studies)

3. خلاص. كل واحد بيطلّع رسالة نجاح من غير ما يمس بياناتك.

ملاحظات:
- لو اشتغل أي ملف قبل كده، عادي خالص تشغّله تاني — مش هيكرر ولا هيمسح.
- ماتشغّلش أي ملفات seed-*.sql (البذور) — دي للمحتوى المبدئي بس وممكن تكتب
  فوق بياناتك الحالية.
