محتاجين نعمل Import Wizard احترافي بدل الاستيراد المباشر، ويكون مخصص حاليًا لـ:

1. فواتير العملاء
2. فواتير الموردين
3. القيود اليومية

الفكرة:
المستخدم يرفع ملف Excel / CSV، والنظام لا يقوم بالحفظ مباشرة. بدل ذلك يفتح شاشة Preview داخلية لمراجعة البيانات وربط الأعمدة وتصحيح الأخطاء قبل التسجيل النهائي.

المطلوب:

## 1. Upload Step

* المستخدم يرفع ملف Excel / CSV.
* لا يتم حفظ أي بيانات مباشرة بعد الرفع.
* يتم الانتقال إلى شاشة Import Wizard.

## 2. Column Mapping

المستخدم يقدر يربط أعمدة الملف مع حقول النظام.

### فواتير العملاء:

* رقم الفاتورة
* تاريخ الفاتورة
* اسم العميل
* العملة
* المنتج أو الخدمة
* الكمية
* السعر
* الخصم
* الضريبة
* الإجمالي قبل الضريبة
* الإجمالي شامل الضريبة
* حساب الإيراد المرتبط
* مركز التكلفة
* المشروع

### فواتير الموردين:

* رقم فاتورة المورد
* تاريخ الفاتورة
* اسم المورد
* العملة
* المنتج أو الخدمة
* الكمية
* السعر
* الخصم
* الضريبة
* الإجمالي قبل الضريبة
* الإجمالي شامل الضريبة
* حساب المصروف / الأصل / المخزون المرتبط
* مركز التكلفة
* المشروع

### القيود اليومية:

* تاريخ القيد
* رقم القيد / المرجع
* شرح القيد
* كود الحساب
* اسم الحساب
* مدين
* دائن
* العملة
* سعر الصرف
* مركز التكلفة
* المشروع
* البيان على مستوى السطر

## 3. Smart Auto Mapping

* النظام يحاول يتعرف تلقائيًا على أسماء الأعمدة الشائعة.
* مثال:
  Date → التاريخ
  Customer → العميل
  Supplier → المورد
  Debit → مدين
  Credit → دائن
  Amount → المبلغ
  Tax → الضريبة
* المستخدم يقدر يعدل الـ Mapping يدويًا.

## 4. Date Format Handling

إضافة اختيار لصيغة التاريخ أثناء الاستيراد:

* Auto Detect
* DD/MM/YYYY
* MM/DD/YYYY
* YYYY-MM-DD
* DD-MM-YYYY
* Excel Serial Date

ويتم عرض التاريخ بعد التحويل في شاشة Preview قبل الحفظ.

## 5. Validation Before Import

لا يتم حفظ أي بيانات قبل مراجعتها.

### Validation لفواتير العملاء:

* العميل موجود أو يتم إنشاؤه حسب اختيار المستخدم.
* رقم الفاتورة غير مكرر.
* العملة صحيحة.
* المنتج أو الخدمة موجودة أو يتم إنشاؤها حسب اختيار المستخدم.
* حساب الإيراد موجود.
* الضريبة صحيحة.
* الإجمالي محسوب بشكل صحيح.

### Validation لفواتير الموردين:

* المورد موجود أو يتم إنشاؤه حسب اختيار المستخدم.
* رقم الفاتورة غير مكرر لنفس المورد.
* الحساب المرتبط موجود.
* الضريبة صحيحة.
* الإجمالي محسوب بشكل صحيح.

### Validation للقيود اليومية:

* الحسابات موجودة في شجرة الحسابات.
* إجمالي المدين يساوي إجمالي الدائن.
* لا يسمح بقيد غير متوازن.
* التاريخ صحيح.
* مركز التكلفة والمشروع صحيحين إن وجدوا.

## 6. Preview & Edit Screen

بعد رفع الملف تظهر شاشة Preview فيها:

* أول 50 صف أو كل البيانات مع pagination.
* كل صف قابل للتعديل قبل الحفظ.
* الصفوف الصحيحة تظهر عادي.
* الصفوف التي بها Warning تظهر باللون الأصفر.
* الصفوف التي بها Error تظهر باللون الأحمر.
* يتم عرض سبب الخطأ في نفس الصف.

## 7. Import Confirmation

قبل الحفظ تظهر Summary:

* إجمالي الصفوف
* الصفوف الصحيحة
* الصفوف التي بها تحذيرات
* الصفوف المرفوضة

الأزرار:

* Import Valid Rows
* Cancel Import
* Download Error Report

## 8. Import History

إضافة سجل للاستيراد يحتوي على:

* اسم الملف
* المستخدم
* نوع الاستيراد
* التاريخ والوقت
* عدد الصفوف
* عدد الصفوف الناجحة
* عدد الصفوف المرفوضة
* Download Error File

## 9. Accounting Posting

بعد تأكيد الاستيراد:

* فواتير العملاء يتم تسجيلها كفواتير مبيعات وتوليد أثرها المحاسبي طبقًا للنظام.
* فواتير الموردين يتم تسجيلها كفواتير مشتريات وتوليد أثرها المحاسبي طبقًا للنظام.
* القيود اليومية يتم تسجيلها فقط إذا كانت متوازنة.

## 10. Important

نحتاج هذا الـ Import Wizard يكون reusable component بحيث نستخدمه لاحقًا في أي موديول جديد، وليس كود منفصل لكل شاشة.

الهدف:
تحويل الاستيراد من مجرد رفع Excel مباشر إلى تجربة احترافية آمنة تقلل الأخطاء وتسمح للمستخدم بمراجعة البيانات وتصحيحها قبل الحفظ.
